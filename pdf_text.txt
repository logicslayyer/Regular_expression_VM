1  Deterministic Finite Automaton (DFA)
Every state has exactly one transition per input symbol
Formal Definition
A DFA is a 5-tuple M = (Q, Σ, δ, q0, F) where:

DFA Rules (must memorise)
 Exactly ONE transition for every (state, symbol) pair — no missing, no ambiguous
 No ε-transitions (cannot move without reading a symbol)
 δ is a total function — defined for all states and all symbols
 Accept if and only if the machine ends in a state q ∈ F after reading the entire input
 Reject if the machine ends in a state q ∉ F (no crashes — always defined)
 Closed under: union, intersection, complement, concatenation, Kleene star

DFA Design Technique: Track what suffix the machine has seen
For a DFA accepting strings ending in pattern P, create states corresponding to how many characters of P
have been matched so far. This is called the suffix automaton construction. The Knuth-Morris-Pratt
failure function tells you which earlier state to fall back to on a mismatch.
DFA Minimization (Myhill-Nerode Theorem)
Two states p, q are equivalent if for all strings w: δ*(p,w) ∈ F ↔ δ*(q,w) ∈ F. The minimization algorithm
(table-filling / Hopcroft) merges equivalent states, producing the unique minimum-state DFA for the
language.
Step
Action
1
Mark all (accept, non-accept) pairs as distinguishable
2
For each unmarked pair (p,q) and each symbol a: if (δ(p,a), δ(q,a)) is marked → mark (p,q)
3
Repeat step 2 until no new pairs are marked
4
Merge all unmarked (equivalent) pairs into single states

2  Nondeterministic Finite Automaton (NFA)
Multiple transitions allowed per symbol; ε-transitions permitted
Formal Definition
An NFA is a 5-tuple N = (Q, Σ, δ, q0, F) where δ: Q × (Σ ∪ {ε}) → P(Q) maps to a power set of states.
NFA Rules
 δ returns a SET of states (possibly empty ∅) — not a single state
 ε-transitions: machine can change state without reading any symbol
 At each step, machine 'branches' into all possible next states simultaneously
 Input accepted if AT LEAST ONE branch of computation ends in an accept state
 Input rejected only when ALL branches fail or die (reach ∅)
 ε-closure(q) = all states reachable from q via ε-transitions alone (including q itself)

3  NFA → DFA Conversion (Subset Construction)
Every NFA can be converted to an equivalent DFA — same language, different structure
Key Insight
An NFA with n states can be simulated by a DFA whose states are subsets of NFA states. In the worst
case the DFA has 2n states, but in practice far fewer are reachable.
Subset Construction Algorithm
 Step 1 — Start: DFA start state = ε-closure({q0})
 Step 2 — For each DFA state S (a set of NFA states) and each input symbol a:
 MOVE(S, a) = ∪{ δ(q, a) | q ∈ S }
 New DFA state = ε-closure(MOVE(S, a))
 Step 3 — Add the new state to the DFA table if not already present
 Step 4 — Mark a DFA state as accept if it contains any NFA accept state
 Step 5 — Repeat until no new DFA states are created
 Step 6 — Dead state ∅ (empty set): add a non-accepting sink state if MOVE returns ∅

4  Regular Expressions → Automata
Thompson's construction: regex → NFA → (subset construction) → DFA
Regular Expression Equivalence Theorem
Every regular expression R describes exactly the same class of languages as DFAs and NFAs. The three
models — RE, NFA, DFA — are all equivalent in expressive power (Kleene's Theorem).
Regular Expression Operators & Rules
 ε — matches the empty string only
 a — matches the single symbol a (for a ∈ Σ)
 R|S — union: matches any string matched by R OR S
 RS — concatenation: matches strings formed by a match for R followed by a match for S
 R* — Kleene star: matches zero or more repetitions of strings matching R
 R+ — one or more: equivalent to RR*
 R? — optional: equivalent to R|ε
 Precedence (high→low): * > concatenation > |
 Parentheses override precedence: (a|b)* means zero or more a's or b's
Thompson's Construction (Regex → NFA)
Each regex operator maps to a small NFA fragment. Combine fragments bottom-up.

PDA Transition Notation & Rules
 Transition written as: (p, a, A) → (q, γ) meaning:
 In state p, read input a (or ε), pop A from stack, push γ, go to state q
 γ = ε means pop without pushing (stack shrinks)
 γ = A means no net change (replace A with A)
 γ = BA means push B on top of A (stack grows)
 Accept by final state: input consumed AND current state ∈ F
 Accept by empty stack: input consumed AND stack is empty
 Both acceptance modes are equivalent (can convert between them)

TM Rules & Key Facts
 Tape is INFINITE in both directions, initially filled with blanks (■)
 Head reads ONE cell at a time, then writes a symbol and moves L or R
 δ(q, a) = (q', b, D): in state q reading a, write b, move direction D, go to q'
 Halting: machine halts only on qaccept or qreject (may loop forever otherwise)
 Accepts: if computation reaches qaccept
 Rejects: if computation reaches qreject OR loops forever (does not halt)
 Decidable language: TM always halts (on every input) — never loops
 Recognisable (RE) language: TM halts and accepts on strings in L; may loop on others
 Church-Turing Thesis: any effectively computable function can be computed by a TM

Common Exam Pitfalls
DFA vs NFA power: Both recognise exactly regular languages — NFA is NOT more powerful, just more convenient
to design
ε-closure: Always compute ε-closure after EVERY move in NFA simulation, not just at the start
Subset construction: The dead state ∅ must be added as a non-accepting DFA state when MOVE returns empty
set
PDA acceptance: By final state and by empty stack are equivalent models — specify which you are using
TM looping: Looping forever ≠ rejecting; only qreject is a rejection. Undecidable languages cause infinite loops
Complement of CFL: CFLs are NOT closed under complement or intersection with another CFL
Pumping Lemma: The lemma can prove a language is NOT regular, but passing the lemma does not prove
regularity
Church-Turing: The thesis is a conjecture / definition, not a theorem — it cannot be formally proved
